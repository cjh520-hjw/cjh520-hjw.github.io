数值范围

总结来说，4个字节占32位的原因是：

  • 一个字节定义为8位。（补充：n位可以表示的最大无符号整数是\(2^n\)-1）
  • 4个字节就是 4×8=32 位。

数据存放

计算机底层用二进制，内存以字节为单位组织的

原码与补码

所有数据在内存中以补码形式存放，而不是原码

正数的补码与原码相同，负数不同：

1、原码就是 “符号位 + 数的绝对值”，即用最高 bit 位表示 “符号”，其余位表示 “数值”。

2、反码的表示方法是：

  • 正数的反码是其本身；
  • 负数的反码是在其原码的基础上，符号位不变，其余各位 “按位取反” 。

3、补码的表示方法是：

  • 正数的补码是其本身；
  • 负数的补码是在其原码的基础上，符号位不变，其余各位 “按位取反 + 1” 。即：在反码的基础上 + 1

-85 1101 0101 1010 1010 1010 1011
9 0000 1001 0000 1001 0000 1001
-9 1000 1001 1111 0110 1111 0111

注意：表中的二进制数只写了 8 位，实际内存不止 8 位。若有 int a=-85; 则对应 32 位存储。

一、机器数与补码存储基础

  1. 存储方式：数值在计算机中以补码形式存储。
  2. 机器数：数在计算机中的二进制表示形式，带符号，最高位存符号（正数为 0，负数为 1 ），但其形式值不等于真正数值，如补码10000010，最高位 1 表负，真实值是 -126 。
  3. 真值：带符号位机器数对应的真正数值。

二、原码、反码、补码的设计逻辑

  1. 问题根源：计算机为简化电路设计，只有加法无减法，靠符号位实现加减法转换（如1 - 1 = 1 + (-1) ），符号位参与运算催生反码。
  2. 反码局限：仅用反码计算会使 0 有0000 0000和1000 0000两个编码，0 带符号无意义，需改进。
  3. 补码解决：负数补码为反码 + 1，正数补码是自身；解决 0 的符号与编码问题，0000 0000表 0，1000 0000表 -128 ，让运算和存储更合理 。

简言之，先讲计算机以补码存数，介绍机器数、真值概念；再解释因计算机运算设计，从原码到反码再到补码的演进，补码解决反码存 0 的弊端，让数值存储和运算更适配计算机体系 。

十进制整数

2468=2×10³+4×10²+6×10¹+8×10⁰

因为这种数字表示法是基于10的幂，所以将它称作基数为10的表示法或十进制表示法。

可以用任何数作基数，如C++允许使用基数8（八进制）和基数16（十六进制）来书写整数。

注意，10⁰为1，任何非零数的0次幂都为1

八进制整数

八进制数是基数为8的，用数字0-7来书写数字。

C++ 用前缀0来表示八进制表示法，如0177 是一个八进制值。

UNIX操作系统常使用八进制来表示值

十六进制整数

十六进制数是基数为16的，这意味着十六进制的 10 表示 16 + 0，即 16

为表示 10 - 15 ，需要字母 a - f。C++ 接受这些字符的大写和小写版本

C++ 使用 0x 或 0X 来指示十六进制表示法，如 0x2B3 是一个十六进制值

硬件文档常使用十六进制来表示诸如内存单元和端口号等值

二进制整数

不管是使用十进制、八进制，还是十六进制表示法来书写整数，计算机都将它存储为二进制值

二进制表示法只使用两个数字 ——0 和 1，每个单元（位）都可以设置成开或关。只是将关表示为 0，将开表示为 1。

但 C++ 没有提供二进制表示法来书写数字的方式。

分析：

  1. C++ 浮点型输出规则

    cout 默认对 float 、double 均输出 6 位有效数字 ，不区分类型；printf 用 %f 格式默认 小数点后 6 位 ，可通过 %.nf 控制小数位数。

    fixed 是输出格式标记（需 <iomanip> 头文件），开启后 cout 切换为 定点表示法 ，默认显示小数点后 6 位，常用来精确控制小数输出位数。

  2. float与double精度差异

    float 通常是 单精度（32 位） ，有效数字约 6 - 7 位；double 是 双精度（64 位） ，有效数字约 15 - 16 位 。赋值、输出时，float 因存储位数少易出现截断，double 能保留更多有效位。

  3. 四舍五入与有效位现象

    当数值有效位接近类型精度上限（如 float 存 7 位有效数时），输出会触发四舍五入；修改整数位长度后，可直观对比 float 因有效位不足导致的截断与 double 精准存储的区别，体现两类浮点型在精度、输出表现上的特性。

实数在 C++ 语句中又称为浮点数，有两种表示方式：

  1. 十进制小数：
  • 十进制形式由数字和小数点组成（必须有小数点）。
  • 单精度末尾加上f，长双精度末尾加上L
  1. 指数：
  • 1e12表示 1*\(10^{12}\)
  • 指数形式 e (或 E) 前必须有数字，且 e 后面必须是整数

在程序中不论浮点数表示为小数形式还是指数形式，内存中都是以 科学计数法 存储的（二进制）：

通用表达式：\(N = S×R^J\)

如：\(490000000000 = 4.9×10^{11}\)、

\(-0.0000000026 = -2.6×10^{-9}\) 数符 + 尾数为 -2.6，阶符 + 阶码 -9

数有正有负，所以设置数符；阶码亦有正负，所以设置阶符；

\(J_t\)（阶符） J（阶码） \(S_f\)（数符） S（尾数）
  • 尾数（S）：决定有效数字，体现数值精度（如 \(4.9×10^{11}\) 里的 \(4.9\) ）。
  • 阶码（J）：指数部分，控制数值范围（如 \(10^{11}\) 里的 11 ）。
  • 基数（R）：可选 2、8、16 等（计算机中常为 2 ，一旦确定机器内固定）。
  • 数符：标记尾数正负（如 \(-2.6×10^{-9}\) 里的负号 ）。
  • 阶符：标记阶码正负（如 \(10^{-9}\) 里的负号 ）。
🫣 int 和 float 同占 4 字节（32 位），但float取值范围差异大的原因

本质差异：存储格式不同
一句话总结：同是 4 字节，int 直接存数值，范围受位数限制；float 用 “符号 + 指数 + 尾数” 的浮点结构，靠指数位大幅扩展了取值范围（但牺牲了整数精度 ）。

  1. int 是直接存整数的二进制值，用全部 32 位表示数值大小与符号
  2. float 是按 “科学计数法（浮点形式）” 存储，32 位拆成三部分:
  • 1 位符号位（第 31 位）：标记正负
  • 8 位指数位（第 30 - 23 位）：控制数值范围
  • 23 位尾数位（第 22 - 0 位）：决定精度（有效数字）

int 用 32 位表示整数，范围是 \(-2^{31}\) ~ \(2^{31} - 1\)（即十进制 -2147483648 ~ 2147483647 ）
指数位实际能表示的指数范围是 \(-2^7\) ~ \(+2^7 - 1\)（即十进制 -128 ~ 127 ）。结合基数（二进制下基数为 2 ），float 取值范围可达到 \(-2^{128}\) ~ \(2^{127}\) ，换算成十进制约 \(\boldsymbol{\pm 3.4×10^{38}}\) 。

规范化的指数形式（科学计数法）：

  • 小数点左边应当只有一位非 0 的数字
  • 一个实数以指数形式输出时，是按照规范化的指数形式输出的。

对比理解：

宏定义写法 替换后式子（简化） 运算结果 关键原因
#define f(z) z*z 4+4*4+4/2+2*2+2（无括号干扰） 28 运算符优先级改变，宏不处理括号
#define f(z) (z)*(z) (4+4)*(4+4)/(2+2)*(2+2)（部分括号） 64 括号改变局部优先级，但整体仍受运算符顺序影响
#define f(z) ((z)*(z)) ((4+4)*(4+4)) / ((2+2)*(2+2))（完全括号） 4 双重括号强制保证 z*z 优先计算，结果正确

C++ 中主要的构造类型包括 struct（结构体）、class（类）、union（联合体），基于这些类型声明的变量均为构造型变量。

二、构造型变量的特点

  1. 复合性：由多个成员（数据或函数）组成，成员可是基本类型（int、float等）或其他构造类型。

  2. 整体性：变量作为一个整体存在，可通过成员访问符（. 或 ->）操作内部成员。

  3. 自定义性：构造类型需先定义（如 struct Person），再声明变量，类型定义决定了变量的结构。

运算符与结合性介绍

  1. 同一优先级的运算符，运算次序由结合方向决定。

    例如，“*” 与 “/” 具有相同的优先级别，其结合方向为自左至右，因此 3 * 5/4 的运算次序是先乘后除。

    例如，负号运算符 “-” 和前置自增运算符 “++” 为同一优先级，结合方向为自右至左，因此 - ++i 相当于 -(++i)。

  2. 不同的运算符要求有不同的运算对象个数。

    例如，加法运算符 “+” 和减法运算符 “-” 为双目运算符，要求在运算符两侧各有一个运算对象（如 3 + 5、8 - 3 等）。

    例如，自增运算符 “++” 和负号运算符 “-” 是一目运算符，只能在运算符的一侧出现一个运算对象（如 - a, i++, –i, (float) i, sizeof (int), *p 等）。

    条件运算符是 C++ 中唯一的一个三目运算符，如 x? a: b。

🫣 浮点数与除法 关键点：另一个操作数也要转换

题目：
(float)(a+b)/2+(int)x%(int)y，设a=2，b=3，x=3.5，y=2.5
答案：
3.5

分析：
根据类型转换规则： 当浮点型（double）与整型（int）进行运算时，整型会自动转换为浮点型，因此实际运算相当于5.0 / 2.0，结果为2.5（double类型）。

运算符的优先级与结合性

算术运算符的结合性是左结合性

各类型数据间的混合运算（隐式转换）

char,short -> int

float -> double

int -> long

int -> double

表达式 10 + 'a' + i * f - d / e 的运算次序如下：

  1. 执行 10 + 'a' ：先把字符 'a' 转换为整数 97，二者相加是 107 。
  2. 执行 i * f：把 i 和 f 都转换成 double 类型，结果为 double 型 。
  3. 执行整数 107 与 i * f 结果相加：把整数 107 转成双精度数（形式为 107.000…00 ），再和 i * f 的结果相加，结果是 double 型 。
  4. 执行 d / e ：把变量 e 转换成 double 型，相除后为 double 型 。
  5. 执行前面相加结果与 d / e 结果相减：用 10 + 'a' + i * f 的结果减去 d / e 的结果，最终结果为 double 型 。
